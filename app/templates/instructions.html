Guide

How To Run And Use The Tracker

This app logs a copy of your prompts after you send them, analyzes grammar in the background, and lets you review patterns later. It never rewrites or blocks the prompt that goes to Codex or Claude Code.

Quick Start

First Run

  1. Start the app with docker compose up --build.
  2. Open the tracker in your browser using the configured host and port.
  3. If APP_API_TOKEN is set, log in with the same token at /login.
  4. If you want LLM analysis, set your own LLM_API_BASE_URL and LLM_MODEL values before starting the app.
  5. Use Manual Capture to verify the dashboard and analysis pipeline before wiring hooks.
Capture

Ways To Send Prompts

  1. Claude Code hook: install the bundled template with python scripts/install_hooks.py --client claude.
  2. Codex hook: install the bundled template with python scripts/install_hooks.py --client codex.
  3. HTTP API: send JSON to POST /api/prompts from any custom script or wrapper.
  4. Import folder: drop .jsonl or .txt files into {{ settings.import_dir }} and scan from Settings.
Claude Code

Project Hook Setup

  1. Run python scripts/install_hooks.py --client claude in the repo root for this project only.
  2. For all Claude Code projects on this machine, use python scripts/install_hooks.py --client claude --scope global instead.
  3. Set PROMPT_TRACKER_TOKEN in your shell if the app uses APP_API_TOKEN.
  4. Confirm that .claude/settings.local.json exists for project-local installs, or ~/.claude/settings.json exists for global installs.
  5. Run /hooks in Claude Code and confirm the hook source is Local for project-local installs or User for global installs.
  6. If you want the installer to write a real tracker host value, set PROMPT_TRACKER_URL before running it. Otherwise edit the generated Claude settings file manually.
  7. Submit a prompt in Claude Code, then open the dashboard to confirm a new prompt appears with source claude-code.

Claude uses the HTTP hook endpoint POST /hooks/claude/user-prompt-submit.

Codex

Command Hook Setup

  1. Run python scripts/install_hooks.py --client codex in the repo root.
  2. Enable codex_hooks = true under [features] in ~/.codex/config.toml.
  3. Set PROMPT_TRACKER_URL and PROMPT_TRACKER_TOKEN if you do not want the forwarder defaults.
  4. Verify that .codex/hooks.json exists and points to scripts/hook_capture_prompt.py.
  5. Restart Codex or open a fresh Codex session in this repo, then send a test prompt.
  6. Open /prompts and confirm the newest prompt appears in the live review section at the top.

This repo's Codex hook path is working on the current Windows setup, but upstream support may still vary across installations.

Daily Use

How To Read The Dashboard

  1. Overview shows average grammar, average clarity, and recurring issue categories.
  2. Prompt History lets you filter by source and latest analysis status, and now pins the newest prompt in a live two-column review.
  3. The live review section shows the original prompt on the left, the suggested rewrite on the right, and highlights the wording changes between them.
  4. Prompt Detail shows the original prompt, suggested rewrite, issue list, and previous analysis runs.
  5. Settings shows runtime config, including whether the LLM grammar analyzer is active, and lets you trigger an import scan manually.
Troubleshooting

Common Problems

  1. No new prompts appear: check the source client hook config first, then verify the app with GET /health.
  2. Dashboard asks for login repeatedly: confirm that the browser token matches APP_API_TOKEN.
  3. Claude hook fails: verify the HTTP endpoint and token header.
  4. Codex hook fails: confirm that Codex was restarted after installing the hook and that the shell launching Codex has the right tracker env vars.
  5. LLM grammar analysis fails: verify ANALYZER_MODE, LLM_MODEL, and that the base URL points to the provider's OpenAI-compatible /v1 endpoint.
  6. Imports stop on malformed JSONL: fix the broken line, then rescan the file.
